初识CiscoAP

2018-06-07 16:14:11 学习记录 阅读量: 0

前几天有人拿来了一个cisco的ap，让我设置成可以自动获取ip的，当时没多想感觉很简单的事情，玩过很多。但是我错了，我想的太简单了。cisco的东西没有那么简单，本来想着通过web直接设置会简单很多。但是没有找到。


百度了一下，发现只有胖ap可以设置web访问，然后我拿到的ap是个瘦ap，具体如何查看这里给大家说一下。


名称的识别：（主要用于在看文档上）

之前老的AP命名规则
LAP： lightweight AP（轻量级AP，也就是我们常说的瘦AP）
AP： 前面不加其他字符，就是胖AP，也就是我们说的Autonomous AP。

目前新的AP命名规则
CAP：Controller-Based AP（基于控制器的AP，也就是瘦AP）
SAP：Standalong AP（独立的AP，也就是胖AP）

通过命令识别

ap#show version



显示如下图：







k9w8为瘦ap、k9w7为胖ap


通过命令发现自己的的ap为瘦ap，好在还可以升级，这里就介绍升级过程了，当然之后我也会把我的升级过程抽时间整理一下，毕竟废了很多时间。


预留一下连接：


然后通过超级终端连接ap设置web访问的命令如下：

ap>enable

显示让输入密码，这个密码在输入的过程中是看不到的，输进去回车就行，默认为Cisco，注意首字母为大写。

ap#

ap#conf t \\回车 备注：conf t 是我写的缩写，如果想看全，按tab键可自动补全

ap...#int bvi1

ap...#ip address 192.168.1.1 255.255.255.0

ap...#no shutdown

ap...#end

ap#show run \\查看有没有成功



到这里就可以通过ip来访问web管理页面了，当然账号密码是默认的Cisco。好了今天就到这里，下一篇抽时间写吧，还有没事百度一下，总有你想要的。
