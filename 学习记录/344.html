Windows Git本地代码推送到github的命令及配置

2018-08-21 15:32:38 字数:9527 阅读量: 500

一、配置

安装git这里就不多说了，个人感觉和安装qq没啥区别。

配置本地信息

git config --global user.name ‘账号名’ 
git config --global user.email 邮箱 
git config --list 查看配置结果

创建本地ssh

$ ssh-keygen -t rsa -C “github邮箱”

打开C:\Users\Administrator.ssh目录，“id_rsa.pub”文件内容就是公钥会用到。

1.登陆github>点击用户头像>Settings>SSH and GPG keys>New SSH key>title:自己看着写>key：打开上面的id_rsa.pub,复制里面的内容粘贴进去>Add SSH key
2.测试ssh key是否成功。
$ssh -T git@github.com
如果出现You’ve successfully authenticated, but GitHub does not provide shell access 。这就表示已成功连上github。

二、Git常用命令

git init   // 初始化版本库

git add .   // 添加文件到版本库（只是添加到缓存区），.代表添加文件夹下所有文件 

git commit -m "first commit" // 把添加的文件提交到版本库，并填写提交备注
git remote add origin 你的远程库地址  // 把本地库与远程库关联

git push -u origin master// 第一次推送时

git push origin master  // 第一次推送后，直接使用该命令即可推送修改
git clone git@github.com:licecn/web1234m.git //Clone with SSH clone到本地
