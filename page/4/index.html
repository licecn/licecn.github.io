    SimpleDateFormat sdf1 = new SimpleDateFormat("yyyy年MM月dd日 HH时mm分ss秒");
    SimpleDateFormat sdf2 = new SimpleDateFormat("yyyy/MM/dd HH:mm");
    SimpleDateFormat sdf3 = new SimpleDateFormat("yyyy-MM-dd HH:mm:ss");

    // 创建Date对象，表示当前时间
    Date now=new Date();

    // 调用format()方法，将日期转换为字符串并输出
    System.out.println(  sdf1.format(now)                        );
    System.out.println(sdf2.format(now));
    System.out.println(sdf3.format(now));

    parse方法

        // 使用parse()方法将文本转换为日期
    String d = "2014-6-1 21:05:36";
    SimpleDateFormat sdf = new SimpleDateFormat("yyyy-MM-dd HH:mm:ss");

     // 调用parse()方法，将字符串转换为日期
    Date date =sdf.parse(d);

    System.out.println(date);

    SimpleDateFormat方法 来设置时间格式。

    Date 类最主要的作用就是获得当前时间，同时这个类里面也具有设置时间以及一些其他的功能，但是由于本身设计的问题，这些方法却遭到众多批评，不建议使用，更推荐使用 Calendar 类进行时间和日期的处理。

    java.util.Calendar 类是一个抽象类，可以通过调用 getInstance()静态方法获取一个 Calendar 对象，此对象已由当前日期时间初始化，即默认代表当前时间，如 Calendar c = Calendar.getInstance();

    其中，调用 Calendar 类的 getInstance() 方法获取一个实例，然后通过调用 get() 方法获取日期时间信息，参数为需要获得的字段的值， Calendar.Year 等为 Calendar 类中定义的静态常量。

    Calendar 类提供了 getTime() 方法，用来获取 Date 对象，完成 Calendar 和 Date 的转换，还可通过 getTimeInMillis() 方法，获取此 Calendar 的时间值，以毫秒为单位

  • 竞价对优化会有影响吗
    李策 SEO优化
    • seo

     2015-12-08 20:44:41

    之前问过很多人，竞价对优化有影响吗？都说没影响，但是心里一直都是犹豫的。应为我接触的很多站都是优化和竞价分离的，所以没有什么证据。

    先说说优化排名的几点：外链、点击、停留时间、内链、二次点击等等吧。 如果说竞价的点击也被列入排名因素的话，那么显而易见对排名是有影响的。在搜索一个词的时候发现一个竞价站的专题页排在了首页，让我很是不解，毕竟这个词还是挺好的一个词。然后点进去看了看，发现就是一个普通的专题页，而且是那种把图片切图，然后通过alt来说明图片的内容。 这么一个页面居然能拍到首页，没有外链，内链几乎没有，那么因素只有点击了，那么点击从哪来，只有竞价了。这个现象不能说竞价占有多大的比重，我只想说竞价其实对优化是有影响的。也许很多人不会同意我的看法吧。当然也没人会拿竞价来做实验。 写这篇文章主要是想表达一下我的想法吧。

    开始的时候只是不能上传图片点击半天没有是吗反映，后来升级了wordpress，发现竟然点击没有反映了。然后我在网上找了找解决办法还真找到了，下面是相关内容。

    可视化/文本模式无法切换，点击添加“媒体按钮”没反应等等。在“优化站Wordpress”的道路上真是一波三折啊，在网上找到了解决办法，特记录下。下面是wordpress的官方解释：

    Disable Javascript Concatenation

    To result in a faster administration area, all Javascript files are concatenated into one URL. If Javascript is failing to work in your administration area, you can try disabling this feature: define(‘CONCATENATE_SCRIPTS’, false);

    汉化其内容大致意思是 wordpress 为了提高效率要求你所有的js要集中到一起，只需更改路径集中存放，问题迎刃而解，具体操作很简单，在wordpress根目录下找到 wp-config.php 这个配置文件，打开文件后在页面的最后添加下面代码:

    define(‘CONCATENATE_SCRIPTS’, false );

    添加之后刷新一下，发现可以用了。 define('CONCATENATE_SCRIPTS', false );
