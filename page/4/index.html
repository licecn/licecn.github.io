  • 【ZT】Java做了6年，给新手一些建议
    李策 Java笔记
    • java
     2016-06-21 14:35:06

    1.不要再拿AWT和Swing练手了，很少公司会用，多写点算法和数据结构，工作后你会获益良多

    2.学Java最重要不是记住基础类型有几种，int的范围是多少，而是培养自己一切皆对象的OO思维，那些所谓的基础，在你实际开发过程中，随着代码量的积累，会变成永久记忆刻在你脑海里的

    3.设计模式和代码重构，Java的两大精髓，一定要重视，要记住结构良好，命名规范的代码，就是最好的注释，很多做了多年java的人只是机械的按照约定俗成的格式写代码，甚至从没试图去搞清楚为什么Service层要加一个interface来实现。

    4.学习框架的时候，不要图快从网上抄大段的配置，去官网或github上找官方文档，照着配，自己做过一次，以后会少很多配置方面的痛苦，相信我

    5.多写多错，多调试bug，每一次通过自己双手而不是依靠网络解决可恶的红字错误，把bug想象成武林高手，每一次bug的解决，都是一次修为的提升。

    6.开个技术博客吧，把自己掌握的技术梳理一遍，把自己的学习心得分享给后来者，利己利人，功德无量

    7.英文很重要，要坚持学

    8.Stay hungry. Stay foolish.保持一颗学习的心，开始的几年，不要太计较薪水差那么几百，5年才是人才和庸才的分水岭，10年才能看出来大神和普通程序员的区别 借用Facebook的一句标语：code wins arguments，别BB，亮代码！

  • Date format()和parse() SimpleDateFormat()
    李策 Java笔记
    • java
     2016-04-21 00:05:22

    format方法

        // 使用format()方法将日期转换为指定格式的文本
    SimpleDateFormat sdf1 = new SimpleDateFormat("yyyy年MM月dd日 HH时mm分ss秒");
    SimpleDateFormat sdf2 = new SimpleDateFormat("yyyy/MM/dd HH:mm");
    SimpleDateFormat sdf3 = new SimpleDateFormat("yyyy-MM-dd HH:mm:ss");

    // 创建Date对象，表示当前时间
    Date now=new Date();

    // 调用format()方法，将日期转换为字符串并输出
    System.out.println(  sdf1.format(now)                        );
    System.out.println(sdf2.format(now));
    System.out.println(sdf3.format(now));

    parse方法

        // 使用parse()方法将文本转换为日期
    String d = "2014-6-1 21:05:36";
    SimpleDateFormat sdf = new SimpleDateFormat("yyyy-MM-dd HH:mm:ss");

     // 调用parse()方法，将字符串转换为日期
    Date date =sdf.parse(d);

    System.out.println(date);

    SimpleDateFormat方法 来设置时间格式。

    Date 类最主要的作用就是获得当前时间，同时这个类里面也具有设置时间以及一些其他的功能，但是由于本身设计的问题，这些方法却遭到众多批评，不建议使用，更推荐使用 Calendar 类进行时间和日期的处理。

    java.util.Calendar 类是一个抽象类，可以通过调用 getInstance()静态方法获取一个 Calendar 对象，此对象已由当前日期时间初始化，即默认代表当前时间，如 Calendar c = Calendar.getInstance();

    其中，调用 Calendar 类的 getInstance() 方法获取一个实例，然后通过调用 get() 方法获取日期时间信息，参数为需要获得的字段的值， Calendar.Year 等为 Calendar 类中定义的静态常量。

    Calendar 类提供了 getTime() 方法，用来获取 Date 对象，完成 Calendar 和 Date 的转换，还可通过 getTimeInMillis() 方法，获取此 Calendar 的时间值，以毫秒为单位

  • 竞价对优化会有影响吗
    李策 SEO优化
    • seo
    • 竞价
     2015-12-08 20:44:41

    之前问过很多人，竞价对优化有影响吗？都说没影响，但是心里一直都是犹豫的。应为我接触的很多站都是优化和竞价分离的，所以没有什么证据。

    先说说优化排名的几点：外链、点击、停留时间、内链、二次点击等等吧。 如果说竞价的点击也被列入排名因素的话，那么显而易见对排名是有影响的。在搜索一个词的时候发现一个竞价站的专题页排在了首页，让我很是不解，毕竟这个词还是挺好的一个词。然后点进去看了看，发现就是一个普通的专题页，而且是那种把图片切图，然后通过alt来说明图片的内容。 这么一个页面居然能拍到首页，没有外链，内链几乎没有，那么因素只有点击了，那么点击从哪来，只有竞价了。这个现象不能说竞价占有多大的比重，我只想说竞价其实对优化是有影响的。也许很多人不会同意我的看法吧。当然也没人会拿竞价来做实验。 写这篇文章主要是想表达一下我的想法吧。

    作为在天朝长大的孩子，有些东西真的能尽善尽美。我伟大的wordpress也未能幸免遇难，很多网站外国网站被墙了，导致wordpress在加载的时候异常的慢。对wordpress进行过优化，打开速度确实有所提升，但是又一个问题砸在我的头上。gravatar也出问题了，导致无法加载，无奈只能通过插件来解决问题，但是很多时候再打开网站的时候有些头像还是打不开，郁闷了。 没办法，不是大神的我只能选择国内的博客程序。 找了很长时间，发现emlog和z-blog 两款博客程序在中国还是可以的，然后我就一个一个的下载下来进行本地测试，这里说一下两个博客的优点和缺点吧。 emlog ，重点在于轻。不管是前台还是后台给人的感觉就是很简洁，非常适合新手或对博客要求不高且不懂代码的人。如果你的服务器不是很好，同时对博客要求不高，可以选择这个。 z-blog ，这个以前是asp版的，现在出了一个php版本的。给人的感觉还可以，插件不是很强大。最重要的是php版本感觉像是后妈生的，还是asp 版本的亲。但是我对asp的印象不是很好所以决然拒绝了z-blog。如果你对博客有点要求但是不是很高，可以推荐给你这个z-blog。 也许是出于装逼de心理，最后还是选择了wordpress。我会继续优化wordpress，到时候 会把优化的内容单独写成日志。
